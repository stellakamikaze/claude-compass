# Nuovo Progetto

Workflow guidato per avviare un nuovo progetto dall'idea al primo commit.

## Fasi

Esegui le fasi in sequenza. Ogni fase produce output che alimenta la successiva.

### Fase 1: Discovery
Esegui `/discovery` per raccogliere requisiti:
- Business requirements
- Technical context
- Scale & operations
- MVP prioritization

### Fase 2: Scope
Dopo la discovery, esegui `/scope` per definire:
- Architettura
- Schema database
- API endpoints
- Componenti chiave

### Fase 3: Plan
Con lo scope definito, esegui `/write-plan` per:
- Ordinare i task
- Definire dipendenze
- Pianificare vertical slices

### Fase 4: Implementazione
Segui il piano creato, implementando feature-by-feature.

### Fase 5: Verifica
Prima del commit, esegui `/arewedone` per:
- Verificare completezza
- Rimuovere dead code
- Commit finale

## Avvia

Inizia con la Fase 1: esegui `/discovery` per raccogliere i requisiti del progetto.
